# Bancos de Dados NoSQL | NoSQL Databases


## Sobre a Disciplina | About the Course

Esta disciplina aborda os principais aspectos dos bancos de dados NoSQL, suas características, modelos, aplicações e benefícios em comparação com os bancos de dados relacionais tradicionais. Com foco em implementações práticas e estudos de caso reais, o curso prepara os estudantes para projetar e implementar soluções de armazenamento adequadas a diferentes cenários de negócio, especialmente aqueles que envolvem grandes volumes de dados.

## Ementa | Course Description

* **Bancos de Dados NoSQL**
  * Definição e características fundamentais
  * Motivações para adoção
  * Modelos de transações em NoSQL

* **Comparativos de Modelos**
  * Propriedades do Modelo Relacional vs. Propriedades dos Modelos NoSQL
  * Teorema CAP e suas implicações

* **Soluções e Implementações**
  * Principais Sistemas Gerenciadores de Bancos de Dados NoSQL
  * Soluções para Big Data
  * Casos de uso e aplicações práticas

## Unidades de Ensino | Course Units

### UNIDADE 1 - Bancos de Dados NoSQL

Esta unidade introduz os conceitos fundamentais de bancos de dados NoSQL, explorando suas características distintivas e os cenários onde oferecem vantagens em relação aos sistemas relacionais tradicionais.

1.1 - Características dos bancos de dados NoSQL
   * Escalabilidade horizontal
   * Distribuição e replicação de dados
   * Flexibilidade de esquema
   * Alta performance para operações específicas

1.2 - Definição de negócios, motivação, tipos e modelos de transações
   * Evolução dos requisitos de armazenamento
   * Limitações do modelo relacional
   * Transações ACID vs. BASE
   * Consistência eventual

1.3 - Tipos de Bancos NoSQL
   * Key-value stores
   * Document databases
   * Column-family stores
   * Graph databases
   * Time-series databases

1.4 - Principais bancos do mercado
   * MongoDB
   * Cassandra
   * Redis
   * Neo4j
   * CouchDB
   * DynamoDB

### UNIDADE 2 - Modelos de SGBD's NoSQL

Esta unidade aprofunda a compreensão dos diferentes modelos NoSQL, com foco especial no MongoDB como exemplo prático de banco de dados orientado a documentos.

2.1 - Modelos relacionais SQL X Modelos NoSQL
   * Estrutura tabular vs. estruturas flexíveis
   * Normalização vs. desnormalização
   * Joins vs. dados aninhados
   * Análise comparativa de performance

2.2 - Propriedades NoSQL
   * Particionamento e sharding
   * Replicação e alta disponibilidade
   * Escalabilidade e distribuição
   * Consistência configurável

2.3 - MongoDB Introdução
   * Arquitetura e componentes
   * Documentos, coleções e bancos de dados
   * BSON e tipos de dados
   * Instalação e configuração

2.4 - MongoDB Operações
   * Shell interativo
   * Índices e performance
   * Agregações e pipelines
   * Replicação e sharding

2.5 - MongoDB CRUD
   * Criação de documentos
   * Consultas e filtros
   * Atualização de dados
   * Remoção de documentos

2.6 - MongoDB Frameworks
   * Mongoose para Node.js
   * PyMongo para Python
   * Drivers oficiais e conexões
   * ODMs (Object Document Mappers)

### UNIDADE 3 – NoSQL e NewSQL para Big Data Analytics e Computação distribuída

Esta unidade explora a aplicação de NoSQL em cenários de Big Data, incluindo análises avançadas, computação distribuída e considerações de segurança.

3.1 - Aplicando NoSQL em BigData, casos de uso e tecnologias
   * Integração com ecossistemas Hadoop
   * Processamento em tempo real com Spark
   * Data Lakes e armazenamento distribuído
   * Análise de casos de sucesso

3.2 - Web Crawlers
   * Coleta de dados não estruturados
   * Persistência de dados web em NoSQL
   * Indexação e busca
   * Processamento em larga escala

3.3 - Segurança e Tipos de Ataques em bancos NoSQL
   * Vulnerabilidades comuns
   * NoSQL injection
   * Controle de acesso e autenticação
   * Criptografia e proteção de dados

### UNIDADE 4 – EXTRA CASSANDRA

Esta unidade oferece uma introdução ao Apache Cassandra, um sistema de banco de dados distribuído altamente escalável.

4.1 - Introdução ao Cassandra
   * Arquitetura descentralizada
   * Modelo de dados e CQL
   * Consistência ajustável
   * Topologia e gerenciamento de clusters

4.2 - Soluções integradas e operações básicas
   * Operações CRUD em Cassandra
   * Modelagem de dados eficiente
   * Integração com ferramentas de análise
   * Monitoramento e performance

## Objetivos e Métodos Didáticos | Objectives and Teaching Methods

A disciplina visa habilitar os alunos a:

* **Analisar cenários complexos de negócios** e escolher a melhor técnica para implementação do banco de dados, objetivando atendimento aos requisitos específicos.
* **Utilizar ferramentas de mercado** que possibilitem a automatização das tarefas de operação e criação das estruturas em ambiente NoSQL.
* **Conhecer e utilizar os recursos básicos** dos principais tipos de SGBDs NoSQL disponíveis no mercado.
* **Concluir a implantação de projetos** de banco de dados em ambiente NoSQL com operação de dados.

## Conexão com o Mercado | Industry Relevance

O conhecimento em bancos de dados NoSQL é altamente valorizado no mercado atual, especialmente para posições como:

* Arquiteto de Dados
* Engenheiro de Dados
* Desenvolvedor Backend
* Especialista em Big Data
* Cientista de Dados
* Analista de Banco de Dados

Empresas de diversos setores utilizam NoSQL para resolver problemas específicos, como:

* E-commerce: catálogos de produtos e recomendações personalizadas
* Redes sociais: armazenamento de conteúdo gerado por usuários
* IoT: captura e análise de dados de sensores
* Finanças: detecção de fraudes em tempo real
* Saúde: armazenamento de registros médicos e análises de dados genômicos
